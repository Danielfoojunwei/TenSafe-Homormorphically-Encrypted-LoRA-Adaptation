## 1. Community Support (Free Tier)

### Overview

Community support is available to all TenSafe users, including those on the free tier. This tier relies on community resources and public documentation.

### Support Channels

#### GitHub Issues
- **URL:** https://github.com/tensafe/tensafe/issues
- **Use for:** Bug reports, feature requests, technical questions
- **Response:** Community volunteers and TenSafe team (best effort)

#### Community Discord
- **URL:** https://discord.gg/tensafe
- **Channels:**
  - `#general` - General discussion
  - `#help` - Technical questions
  - `#showcase` - Share your projects
  - `#announcements` - Product updates
- **Moderation:** Community moderators + TenSafe team

#### Documentation
- **URL:** https://docs.tensafe.io
- **Contents:**
  - Getting started guides
  - API reference
  - Tutorials and cookbooks
  - Troubleshooting guides

#### Stack Overflow
- **Tag:** `tensafe`
- **Use for:** Technical questions with code examples

### Response Expectations

| Channel | Typical Response |
|---------|-----------------|
| GitHub Issues | 2-5 business days |
| Discord | Hours to days (community) |
| Stack Overflow | Varies (community) |

### Limitations

- No guaranteed response times
- No direct access to TenSafe engineers
- No phone or private support
- Security issues should be reported privately (security@tensafe.io)

## Support Request Guidelines

### How to Submit a Support Request

#### Via Email
```
To: [appropriate support address]
Subject: [Severity] - Brief description

Body:
- Account/Organization ID
- Environment (production/staging)
- Affected service(s)
- Error messages/logs
- Steps to reproduce
- Business impact
- Workaround attempted
```

#### Via Support Portal
1. Log in to https://support.tensafe.io
2. Click "New Ticket"
3. Select severity level
4. Fill in required fields
5. Attach relevant logs/screenshots
6. Submit

### Severity Selection Guide

| Select This | When |
|-------------|------|
| P1 - Critical | Complete outage, data loss risk, security breach |
| P2 - Major | Major feature unavailable, significant degradation |
| P3 - Minor | Minor issue, workaround available |
| P4 - Low | Questions, feature requests, documentation |

### Information to Include

**Always Include:**
- Account identifier
- Environment details
- Timestamp of issue
- Error messages

**For Technical Issues:**
- API endpoint affected
- Request/response samples
- Logs (sanitized of sensitive data)
- Steps to reproduce

**For Performance Issues:**
- Metrics/graphs
- Baseline comparison
- Recent changes

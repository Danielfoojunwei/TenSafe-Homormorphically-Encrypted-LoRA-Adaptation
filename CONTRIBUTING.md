# Contributing to TenSafe

Thank you for your interest in contributing to TenSafe! We welcome contributions from the community to help make privacy-preserving machine learning more accessible and performant.

## Development Process

1. **Fork and Clone**: Fork the repository and clone it to your local machine.
2. **Environment Setup**: Run `make dev` to install development dependencies.
3. **Branching**: Create a new branch for your feature or bugfix.
4. **Code Quality**: Ensure your code follows the project's style and passes all linting checks.
5. **Testing**: Add tests for new features and ensure all existing tests pass (`make test`).
6. **PR Submission**: Submit a Pull Request with a clear description of your changes.

## Security Policy

If you discover a security vulnerability, please do NOT open a public issue. Instead, report it privately according to our [Security Policy](SECURITY.md).

## Areas for Contribution

- **HE Microkernel**: Performance optimizations for specific hardware backends.
- **Privacy Algorithms**: Improving DP-SGD efficiency or HE-LoRA convergence.
- **Integrations**: Adding support for more ML frameworks or cloud providers.
- **Documentation**: Improving guides, examples, and API references.
